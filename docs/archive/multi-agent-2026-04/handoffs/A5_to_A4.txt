[A5 -> A4 DOC HANDOFF]
New residual audit mode:
- ResidualComputer(..., lap_tau_mode="mixed_legacy")

Supported mode names:
- mixed_legacy
- stretched_divergence

Default behavior:
- default remains mixed_legacy
- trainer / runner / CLI behavior is unchanged because no public config or CLI switch was added

Meaning of the two modes:
- mixed_legacy: stretched grad_tau transport term + unstretched lap_tau term
- stretched_divergence: A5 audit candidate that applies stretched divergence to the safe grad_tau reconstruction

Key conclusion to document:
- stretched_divergence is useful as an audit mode, but it should not replace mixed_legacy as the project default yet
- reason: on debug + smooth_lens it is self-consistent under its own operator, but differs from legacy by about 0.51 relative L2 even on evaluation/physical regions, not just inside PML

Reference audit numbers:
- rel_l2_eval = 0.5084495064331273
